## Marco global
_Responde: como se organiza la vista global de arquitectura y en que orden debe recorrerse._

La vista `arc42` concentra las decisiones estructurales del sistema:
drivers, restricciones, contexto, estrategia, bloques, runtime,
despliegue, conceptos transversales, ADR, escenarios de calidad y deuda.

## Criterio de autoridad
_Responde: como se relaciona esta vista global con producto, dominio y el resto del pilar._

`arc42` no redefine necesidad ni semantica. Parte del baseline vigente
de producto y dominio para fijar realizacion tecnica, decisiones
estructurales y riesgos aceptados del ciclo.

## Convenciones de uso
_Responde: que reglas editoriales gobiernan esta vista global para evitar duplicacion y autoridad difusa._

- las decisiones abiertas del pilar se concentran en `Decisiones Arquitectonicas`;
- las referencias a trabajo historico no se usan como evidencia activa distribuida;
- las capacidades futuras deben marcarse como `reservado para evolucion` si no forman parte del baseline operativo de `MVP`.
