## Cobertura de temas NFR con impacto semantico
| Tema requerido | Cobertura explicita en NFR-007 |
|---|---|
| Restricciones de negocio con impacto operativo | Protege la capacidad operativa de detectar y responder incidentes en flujos core. |
| Condiciones de rendimiento o disponibilidad que alteran diseno | Aplica parcialmente: observa latencia y error rate para sostener objetivos operativos de servicios core. |
| Aislamiento, regionalizacion, seguridad, compliance, trazabilidad | Aporta trazabilidad tecnica (logs/metricas/trazas); no define regionalizacion ni aislamiento de datos como foco principal. |
| NFR que cambia la manera en que el sistema debe ser concebido | Cambia el diseno al exigir estandar de observabilidad como condicion base antes de ampliar alcance funcional. |

## Detalle de la restriccion (tabla)
| Tema | Detalle |
|---|---|
| Enunciado | Observabilidad operacional minima. |
| Metrica y umbral | 100% de respuestas mutantes incluyen `traceId`; alertas para error rate y latencia p95 en servicios core. |
| Verificacion y validacion | Pruebas de contratos de observabilidad + dashboards/alertas. |
| Alcance operativo | Aplica a endpoints y operaciones core del MVP. |
| No alcance | No cubre analitica de negocio avanzada sobre telemetria. |
| Supuestos y restricciones | La observabilidad minima debe estar activa antes de ampliar alcance funcional. |
| Preguntas abiertas | No hay preguntas abiertas bloqueantes para `MVP`; el set minimo de observabilidad queda definido como baseline y su ampliacion es `No bloqueante`. |
| Impacto esperado | Estandar de logs/metricas/trazas y tableros operativos minimos. |
