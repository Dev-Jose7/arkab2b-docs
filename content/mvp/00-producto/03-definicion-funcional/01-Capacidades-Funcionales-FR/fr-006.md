## Detalle del requisito (tabla)
| Tema | Detalle |
|---|---|
| Descripcion | El sistema informa al cliente cambios de estado relevantes del pedido. |
| Racional | Disminuye incertidumbre post-compra y consultas manuales. |
| Alcance | - Notificacion al cliente en cambios de estado clave del pedido.<br>- Registro de intento y resultado de envio. |
| No alcance | - Garantia de entrega de canal externo mas alla del intento registrado.<br>- Automatizaciones de marketing o journeys multicanal avanzados. |
| Actor(es) | - Sistema de pedidos (emisor) y cliente B2B (receptor). |
| Disparador | - Cambio de estado del pedido a un estado contractual definido. |
| Precondiciones | - Pedido existente con transicion de estado valida.<br>- Canal de notificacion configurado para la organizacion. |
| Flujo principal | 1. El sistema detecta transicion de estado relevante.<br>2. Construye mensaje con informacion del pedido y nuevo estado.<br>3. Intenta envio por canal definido.<br>4. Registra resultado del intento. |
| Postcondiciones | - Cliente informado cuando el envio es exitoso.<br>- Trazabilidad de notificaciones disponible para seguimiento. |
| Supuestos y restricciones | - Fallas de notificacion no deben bloquear el flujo transaccional del pedido.<br>- Contratos de integracion de canal se detallan fuera de este pilar. |
| Preguntas abiertas | - No hay preguntas abiertas bloqueantes para `MVP`; la politica fina de timeout/retry por canal queda como `No bloqueante` y se ajusta en operacion. |
| Criterios de aceptacion | - Estados minimos contractuales del baseline `MVP`: `PENDING_APPROVAL`, `CONFIRMED`, `CANCELLED`.<br>- Estados de fulfillment (`READY_TO_DISPATCH`, `DISPATCHED`, `DELIVERED`) quedan reservados para evolucion y no forman parte del baseline operativo actual.<br>- Envio por canal definido (correo/plataforma).<br>- Registro de intento y resultado de notificacion. |
| Dependencias | - FR-004<br>- NFR-007 |
| Riesgos | - Falla en canal de notificacion. |
| Notas de implementacion | No debe bloquear flujo transaccional del pedido. |
