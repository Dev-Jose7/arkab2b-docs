## Detalle del requisito (tabla)
| Tema | Detalle |
|---|---|
| Descripcion | El cliente puede ajustar el pedido mientras su estado lo permita. |
| Racional | Reduce errores y friccion de compra recurrente. |
| Alcance | - Modificacion de pedido en estados permitidos (`PENDING_APPROVAL`).<br>- Recalculo funcional de disponibilidad al cambiar lineas. |
| No alcance | - Cambios sobre pedidos confirmados o en etapas posteriores.<br>- Renegociacion comercial compleja de pedido fuera del flujo MVP. |
| Actor(es) | - Comprador B2B propietario del pedido. |
| Disparador | - Solicitud de editar pedido antes de su confirmacion. |
| Precondiciones | - Pedido existente y asociado a la organizacion del actor.<br>- Estado del pedido dentro de la ventana permitida. |
| Flujo principal | 1. El actor solicita agregar/quitar/ajustar lineas.<br>2. El sistema valida estado permitido para modificar.<br>3. Aplica cambios validos y recalcula disponibilidad.<br>4. Si el estado no permite cambios, retorna `transicion_estado_invalida` sin efectos. |
| Postcondiciones | - Pedido actualizado solo si cumple reglas de estado.<br>- Integridad funcional mantenida sin side effects en rechazos. |
| Supuestos y restricciones | - Las transiciones de estado son parte del contrato funcional del MVP.<br>- El detalle de maquina de estados tecnica se define en otro pilar. |
| Preguntas abiertas | - No hay preguntas abiertas bloqueantes para `MVP`; la ventana exacta de bloqueo previo a confirmacion queda `Parametrizada` y no altera la regla funcional vigente. |
| Criterios de aceptacion | - Solo permite cambios cuando el pedido esta en `PENDING_APPROVAL`.<br>- `CREATED` se considera estado interno transitorio de `order` y no estado contractual observable del MVP.<br>- Recalcula disponibilidad cuando se elimina o reduce una linea.<br>- Si el estado es `CONFIRMED` o superior, rechaza con `transicion_estado_invalida` sin side effects. |
| Dependencias | - FR-004<br>- FR-002 |
| Riesgos | - Cambios tardios impactan despacho. |
| Notas de implementacion | Regla de estado definida para eliminar ambiguedad de testeo. |
