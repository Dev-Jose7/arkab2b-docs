## Detalle del requisito (tabla)
| Tema | Detalle |
|---|---|
| Descripcion | El sistema permite ajustar stock por SKU y evita valores negativos. |
| Racional | Controla sobreventa y errores operativos. |
| Alcance | - Ajuste de stock por SKU con control de no-negatividad.<br>- Registro funcional de actor, fecha y motivo del ajuste. |
| No alcance | - Automatizacion de abastecimiento a proveedores.<br>- Reglas de reserva transaccional detalladas por dominio. |
| Actor(es) | - Operador de inventario o administrador. |
| Disparador | - Solicitud de ajuste de inventario por operacion manual o correccion. |
| Precondiciones | - SKU existente y activo.<br>- Usuario con permiso para operar inventario. |
| Flujo principal | 1. El actor indica SKU, cantidad y motivo de ajuste.<br>2. El sistema valida que el resultado no sea negativo.<br>3. El sistema registra el movimiento con metadatos de auditoria.<br>4. El sistema recalcula disponibilidad del SKU. |
| Postcondiciones | - Stock actualizado sin violar la regla de no-negatividad.<br>- Disponibilidad recalculada para procesos de venta. |
| Supuestos y restricciones | - Los ajustes se ejecutan en contexto de operacion controlada.<br>- El detalle de consistencia transaccional se define en otros pilares. |
| Preguntas abiertas | - No hay preguntas abiertas bloqueantes para `MVP`; el tuning fino de reserva/liberacion bajo picos queda como `No bloqueante` y se cierra en `04-operacion`. |
| Criterios de aceptacion | - No se permite stock negativo.<br>- Cada ajuste queda con fecha, actor y motivo.<br>- Disponibilidad se recalcula tras cada ajuste. |
| Dependencias | - FR-001<br>- NFR-004 |
| Riesgos | - Ajustes manuales sin disciplina operacional. |
| Notas de implementacion | Debe alinear reglas de reserva en dominio. |
