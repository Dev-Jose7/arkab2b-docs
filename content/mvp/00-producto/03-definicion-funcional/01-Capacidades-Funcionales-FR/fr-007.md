## Detalle del requisito (tabla)
| Tema | Detalle |
|---|---|
| Descripcion | El sistema consolida ventas semanales para seguimiento comercial. |
| Racional | Habilita decisiones de negocio y control de crecimiento. |
| Alcance | - Consolidacion semanal de ventas para seguimiento comercial.<br>- Exportacion de resultados en CSV/PDF. |
| No alcance | - Analitica predictiva o modelos de forecasting.<br>- Dashboard en tiempo real con actualizacion continua. |
| Actor(es) | - Administrador comercial u operador de reportes. |
| Disparador | - Corte semanal de negocio o solicitud manual autorizada. |
| Precondiciones | - Datos de pedidos/ventas disponibles para el periodo.<br>- Parametros de corte semanal definidos. |
| Flujo principal | 1. El sistema agrega ventas del periodo semanal.<br>2. Calcula top de productos y clientes frecuentes.<br>3. Genera reporte en formato exportable.<br>4. Publica reporte para consumo operacional. |
| Postcondiciones | - Reporte semanal disponible y trazable por fecha/corte.<br>- Soporte funcional para decisiones comerciales. |
| Supuestos y restricciones | - El valor del reporte depende de la completitud de datos de ventas.<br>- Parametros por pais pueden cambiar el corte operativo. |
| Preguntas abiertas | - No hay preguntas abiertas bloqueantes para `MVP`; el calendario exacto por pais/organizacion queda `Parametrizado` por configuracion regional vigente. |
| Criterios de aceptacion | - Incluye total ventas, productos mas vendidos y clientes frecuentes.<br>- Periodicidad semanal.<br>- Exportable CSV/PDF. |
| Dependencias | - FR-004<br>- NFR-002 |
| Riesgos | - Datos incompletos por carga tardia. |
| Notas de implementacion | Definir cortes de semana operativa por pais. |
