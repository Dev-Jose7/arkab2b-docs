## Detalle del requisito (tabla)
| Tema | Detalle |
|---|---|
| Descripcion | El sistema identifica carritos sin conversion y habilita recordatorio al cliente. |
| Racional | Aumenta conversion y recuperacion de oportunidades. |
| Alcance | - Identificacion de carritos abandonados y habilitacion de recordatorio.<br>- Prevencion de recordatorios duplicados en ventana definida. |
| No alcance | - Campanas de retargeting multietapa.<br>- Segmentacion avanzada de marketing automatizado. |
| Actor(es) | - Operador comercial (consulta/envio) y cliente B2B (receptor). |
| Disparador | - Inactividad del carrito durante la ventana de abandono configurada. |
| Precondiciones | - Carrito con items sin conversion a pedido.<br>- Datos de contacto disponibles para recordatorio. |
| Flujo principal | 1. El sistema identifica carritos sin conversion.<br>2. Muestra listado con fecha y contenido.<br>3. El operador activa recordatorio cuando aplica.<br>4. El sistema valida deduplicacion por ventana y registra el intento. |
| Postcondiciones | - Carritos abandonados quedan trazados para recuperacion comercial.<br>- Se evita saturacion por envios duplicados. |
| Supuestos y restricciones | - El criterio exacto de abandono depende de una ventana parametrizable.<br>- El canal de envio utiliza capacidades de notificacion del MVP. |
| Preguntas abiertas | - No hay preguntas abiertas bloqueantes para `MVP`; el valor de la ventana de abandono queda `Parametrizado` y su tuning por segmento pasa a `Evolucion posterior`. |
| Criterios de aceptacion | - Lista de carritos abandonados con fecha y contenido.<br>- Permite enviar recordatorio.<br>- Evita duplicidad de recordatorios dentro de ventana definida. |
| Dependencias | - FR-004<br>- FR-006<br>- NFR-007 |
| Riesgos | - Recordatorio fuera de contexto comercial. |
| Notas de implementacion | Ventana de abandono cerrada como parametro operativo del baseline; la segmentacion avanzada por tipo de cliente queda para evolucion posterior. |
