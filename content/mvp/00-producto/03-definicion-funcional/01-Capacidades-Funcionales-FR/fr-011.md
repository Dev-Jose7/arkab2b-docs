## Detalle del requisito (tabla)
| Tema | Detalle |
|---|---|
| Descripcion | El sistema permite parametrizar reglas operativas por pais de la organizacion para soportar Colombia y readiness de Ecuador, Peru y Chile sin bifurcar el core. |
| Racional | La expansion regional requiere consistencia funcional con variaciones locales controladas. |
| Alcance | - Parametrizacion operativa por pais para organizaciones del MVP.<br>- Aplicacion de moneda, corte semanal y politicas de retencion por pais.<br>- Bloqueo funcional de operaciones cuando falte configuracion requerida. |
| No alcance | - Implementacion legal exhaustiva por jurisdiccion en esta fase.<br>- Motor regulatorio externo o actualizacion automatica normativa. |
| Actor(es) | - Administrador de plataforma y operador de cumplimiento/operaciones. |
| Disparador | - Alta/edicion de configuracion por pais o ejecucion de operacion dependiente de politica pais. |
| Precondiciones | - Organizacion con pais operativo asignado.<br>- Configuracion por pais disponible y versionada. |
| Flujo principal | 1. El actor configura parametros operativos por pais.<br>2. El sistema versiona y audita cambios con actor/fecha/motivo.<br>3. En runtime, el sistema resuelve politicas por pais para cada organizacion.<br>4. Si no existe configuracion requerida, bloquea con error trazable. |
| Postcondiciones | - Operaciones usan parametros del pais correspondiente sin fallback no autorizado.<br>- Cambios y decisiones quedan auditables. |
| Supuestos y restricciones | - El MVP cubre consistencia operativa regional, no totalidad regulatoria.<br>- El detalle juridico fino queda para iteraciones posteriores. |
| Preguntas abiertas | - No hay preguntas abiertas bloqueantes para `MVP`; el readiness legal exhaustivo por pais queda `Fuera de alcance` del baseline y como `Evolucion posterior`. |
| Criterios de aceptacion | - Cada organizacion opera con parametros de su pais (moneda, corte semanal y politicas de retencion aplicables) sin hardcode por servicio.<br>- La configuracion por pais es versionable y auditable con `actor`, `fecha` y motivo.<br>- Si no existe configuracion del pais, la operacion se bloquea con error de negocio trazable. |
| Dependencias | - FR-003<br>- FR-007<br>- FR-009<br>- NFR-010 |
| Riesgos | - Reglas regulatorias incompletas por pais pueden frenar despliegue regional. |
| Notas de implementacion | Requisito obligatorio en MVP; detalle legal fino por pais permanece como control operativo posterior. |
